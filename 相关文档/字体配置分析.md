我把目录、Excel、ExcelDB 和 IL2CPP dump 都过了一遍。先说结论：**没有可在 ExcelDB 里改的"全局字体"开关**，但我找到了真正的字体配置位置和一种可行的替换思路。

## 为什么会显示生硬

游戏的 `Language` 枚举只有 5 个值（[dump.cs:359944-359953](vscode-webview://0g33afsh4vtoocm85eirbmsl04vmiqc3j79oi84bsaqlno69l5ur/dump.cs#L359944-L359953)）：



```
Kr=0, Jp=1, Th=2, Tw=3, En=4   // 没有 Cn
```

字体管理器 [FontContainer (dump.cs:324803-324839)](vscode-webview://0g33afsh4vtoocm85eirbmsl04vmiqc3j79oi84bsaqlno69l5ur/dump.cs#L324803-L324839) 也只对应 5 个 `[SerializeField] FontLocalizeInfo`：`Kr / Jp / Th / Tw / En`。

也就是说，**你的简中汉化目前是被当作 `Tw` 渲染的**，而 Tw 用的是 **NotoSansTC**（思源黑体繁体版）。简体字用繁体字形渲染就会出现「生硬、字形不对、缺字」的现象——这是根因，不是字体本身难看。

## 字体资产实际在哪

字体完全不在 Excel/ExcelDB 里，而在 Unity Addressables bundle 里。从 `catalog_Windows.bytes` 里提取出的关键信息：

**实际字体文件**（位于 `_MX/UIs/_MXCommon/Fonts/`）：

| 用途                   | 文件                                                         |
| ---------------------- | ------------------------------------------------------------ |
| Tw（繁中，你现在用的） | `NotoSansTC-Medium.otf` / `NotoSansTC-Bold.otf`              |
| Kr（韩）               | `MALGUN.ttf` / `MALGUNBD.ttf`                                |
| Jp（日）               | `MPLUS1p-Medium.ttf` / `MPLUS1p-Bold.ttf`                    |
| En/Th                  | `NotoSans-Regular.ttf` / `NotoSans-Bold.ttf`、`RSU_Regular/BOLD.ttf` |
| 主字体                 | `MainFont.otf` / `MainFont_Bold.otf`                         |

**对应的 bundle 文件**（在 `BlueArchive_Data/StreamingAssets/PUB/Resource/GameData/Windows/`）：

- `assets-_mx-uis-_mxcommon-_mxdependency-fonts-2026-04-06_assets_all_68077459.bundle` ← 字体主包，最关键
- `assets-_mx-uis-_mxcommon-.ttf-_mxdependency-2025-02-11_assets_all_646201891.bundle`
- `assets-_mx-uis-_mxcommon-.ttf-_mxdependency-2025-07-25_assets_all_3705917133.bundle`

- `assets-_mx-uis-_mxcommon-.fontsettings-_mxdependency-2025-07-25_assets_all_3037143754.bundle`

## 三种可行方案

**方案 A（最推荐）：替换 bundle 里的 OTF 文件** 用 AssetStudio / UnityPy 把 `assets-_mx-uis-_mxcommon-_mxdependency-fonts-...bundle` 解包，把 `NotoSansTC-Medium.otf` 和 `NotoSansTC-Bold.otf` 换成简体字体（如 `NotoSansSC-Regular.otf`/`NotoSansSC-Bold.otf`，思源黑体简体），保持文件名不变重新打包。游戏不知道你换了字体，仍然按 "Tw" 加载，但字形会变成简体。**和汉化本身互不干扰**。

**方案 B：替换 `FontContainer` prefab**
 解开挂载 `FontContainer` 的 prefab（地址 `ui/font/bundlefonttw`，GUID `642afebe4bcfc0a41ad5f167d4d552ca`），改 `Normal.FontInfo` / `Bold.FontInfo` 的 Font 引用。比方案 A 多此一举，不推荐。

**方案 C：IL2CPP 层加 `Cn` 语言**
 用 Il2CppDumper 改 `Language` 枚举、扩 `FontContainer` 字段、改 GameAssembly.dll。改动面太大，不推荐。
